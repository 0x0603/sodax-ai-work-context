# Knowledge

Synthesized, reusable knowledge that outlives any single issue — how a subsystem
works, recurring gotchas, conventions discovered, "how to do X in sodax".

Distinct from:

- `research/` — exploratory spikes and option comparisons (the "looking into it"
  trail).
- an issue's `process.md` — notes tied to one task.

When a lesson from an issue is worth keeping long-term, lift it here.

- One file per topic: `kebab-title.md`.
- Group by area with subfolders if it grows (e.g. `knowledge/frontend/...`).
