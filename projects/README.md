# Project Context

Repo-scoped context lives here. Directory names must match the real workspace
repo folder names so agents can navigate by convention:

| Context path | Workspace repo |
| ------------ | -------------- |
| `projects/sodax-backend/` | `sodax-backend/` |
| `projects/sodax-frontend/` | `sodax-frontend/` |
| `projects/sodax-sdks/` | `sodax-sdks/` |
| `projects/intents-whitelabel/` | `intents-whitelabel/` |

Use each project folder for repo-local plans, decisions, and notes. Keep
cross-cutting material in top-level `plans/`, `decisions/`, or `research/`.
